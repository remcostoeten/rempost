Step-by-step breakdown.

The important invariant is simple: the raw email is stored before any parser touches it. Everything else is derived from that canonical record.

1
Email is sent
Someone, or your Hotmail forwarding rule, sends an email to trash-email-a@remcostoeten.nlxxl@remcostoeten.nl. That address does not need a mailbox because Cloudflare intercepts mail for the domain.
2
Cloudflare catches it
Email Routing checks its rules and delivers matching mail to rempost-email-worker-devrempost-email-worker. The Worker is the destination, so no inbox polling is involved.
3
Worker decodes the message
email-worker.mjs receives the raw stream and uses postal-mime to extract sender, subject, text body, HTML body, and headers into a JSON-safe shape.
4
Worker POSTs to Phoenix
The Worker sends JSON to cadillac-jay-cruises-grades.trycloudflare.com/api/inbound/email your-prod-domain/api/inbound/email with x-rempost-token for authentication.
5
Phoenix stores first
InboundEmailController validates the token, creates an InboundEmail row, and only then schedules an EmailParserWorker Oban job.
6
Parser derives structure
Rempost.Parsing.Pipeline runs deterministic parsers for order IDs, carriers, tracking numbers, and shipment status before creating shipment and tracking records.
7
LiveView updates operators
Phoenix PubSub and LiveView push the new operational state to active dashboards. Operators see the shipment appear without refreshing the page.

Cloudflare jargon, plain English.

The pipeline has a few infrastructure-specific names. Each one maps to a concrete job in the ingestion system.

Cloudflare Worker
A small JavaScript program that runs on Cloudflare's network. You deploy it with wrangler deploy, and Cloudflare invokes it when routed email arrives.
Email Routing
Cloudflare's system for intercepting mail for your domain and delivering it to a destination, including a Worker. In Rempost, it replaces inbox querying.
Wrangler Secret
A sensitive runtime value stored encrypted in Cloudflare. The Worker reads it as env.NAME; it should not live in wrangler.toml.
Cloudflare Tunnel
cloudflared gives the local Phoenix server a public HTTPS URL so the deployed Worker can reach localhost:4000 during development.
wrangler.toml
The Worker config file. It defines the script, deployment name, variables, and environment overrides such as [env.dev].
postal-mime
An npm library that converts raw MIME email into usable fields: sender, subject, text body, HTML body, attachments, and headers.
Oban
A background job library for Elixir and Phoenix. Rempost returns quickly from HTTP requests and lets Oban do parser work asynchronously.
LiveView
Phoenix's server-rendered real-time UI. It keeps a socket open and pushes dashboard updates when new derived shipment state is available.
Shared token
A long random value known by the Worker and Phoenix. Phoenix rejects inbound requests that do not include it in x-rempost-token.

Operational by design.

This page documents the ingestion pipeline as a system of record, not as an email client. Raw email comes first; structured intelligence is derived after persistence.
